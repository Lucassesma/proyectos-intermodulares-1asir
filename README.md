# 🚀 Proyecto Intermodular - Infraestructura TI Empresarial

Bienvenido al repositorio del **Proyecto Intermodular de 1º de ASIR**. El proyecto Intermodular es una propuesta integral para el diseño, despliegue y documentación de una infraestructura tecnológica completa para una empresa.

## 📝 Descripción del Proyecto
Este proyecto simula la creación de la infraestructura IT para sesmAI : Una empresa de investigación tecnológica. El objetivo es integrar los conocimientos de todos los módulos del primer año para ofrecer una solución profesional, escalable y segura.


---

## 🏗️ Arquitectura del Sistema

### 🌐 Redes (0370)
Se ha diseñado una topología de red que incluye:
* **Segmentación**: Uso de VLANs para separar el tráfico de administración y usuarios.
* **Direccionamiento**: Plan IP lógico y organizado.
* **Servicios**: Configuración de DHCP, DNS y servicios de archivos.

### 💾 Gestión de Bases de Datos (0372)
La base de datos centraliza la información de [usuarios/productos/reservas]. Incluye:
* **Diseño E/R**: Estructura relacional optimizada.
* **Seguridad**: Gestión de usuarios y permisos específicos.
* **Scripts**: Creación, inserción y consultas complejas en SQL.

### 🖥️ Sistemas Operativos y Hardware (0369/0371)
* **Sistemas**: Despliegue de servidores (Linux/Windows) y equipos cliente configurados.
* **Hardware**: Selección justificada de componentes (CPU, RAM, Almacenamiento SSD) para garantizar el rendimiento.

### 📄 Lenguajes de Marcas (0373)
Gestión de información estructurada mediante:
* **XML**: Representación de datos reales del sistema.
* **XSD**: Validación estricta de la estructura y tipos de datos.

---

## 📁 Estructura del Repositorio
Siguiendo las instrucciones del proyecto, el repositorio se organiza de la siguiente manera:

* `/redes`: Documentación de topología y direccionamiento IP.
* `/bbdd`: Scripts SQL, diagramas E/R y análisis de datos.
* `/sistemas`: Plan de implantación y configuración de SO.
* `/hardware`: Análisis y propuesta de componentes físicos.
* `/xml`: Documentos XML, esquemas XSD y evidencias de validación.
* `/docs/empleabilidad`: Perfil profesional, investigación del sector y portfolio.
* `/docs/cloud`: Propuesta de arquitectura en la nube y estimación de costes.

## 🛠️ Tecnologías Utilizadas
* **Redes**: Cisco Packet Tracer.
* **BBDD**: MySQL.
* **Sistemas**: VirtualBox.
* **Marcas**: XML, XSD, Markdown.
* **Otros**: Git, GitHub, Canva.
